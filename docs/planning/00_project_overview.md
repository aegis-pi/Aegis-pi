## 문제 정의

기존 Safe-Edge는 단일 공장, 폐쇄망, 라즈베리파이 3노드 K3s 기준선으로는 의미 있는 성과를 냈다. 그러나 다음 한계가 있었다.

1. 여러 공장을 중앙에서 함께 보는 운영 구조가 없다.
2. 데이터 보존은 가능하지만 공장 단위 위험 상태를 복제해 보여주는 상위 관제가 없다.
3. 로컬 GitLab + ArgoCD 중심 운영은 외부 연계형 CI/CD와 멀티 환경 Fleet 운영으로 바로 이어지지 않는다.

## 해결 방향

Aegis-Pi는 아래 방향으로 Safe-Edge를 확장한다.

- `factory-a`에 Safe-Edge 기준선을 재구성한다.
- `factory-b`, `factory-c`를 테스트베드형 Spoke로 추가한다.
- AWS EKS Hub에서 여러 Spoke를 중앙 배포/관제한다.
- IoT Core -> S3 -> Risk Score 처리 흐름으로 공장별 위험 상태를 만든다.
- 본사 관제 담당자가 여러 공장의 상태를 한눈에 보는 운영 관제 서비스를 지향한다.

## 대상 사용자

- 1차 사용자: 본사 관제 담당자
- 2차 사용자: 운영/배포 담당 개발자 또는 시스템 관리자
- 후속 사용자: 발표/검토/지도용 이해관계자
