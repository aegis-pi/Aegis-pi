# MVP 범위

상태: source of truth
기준일: 2026-04-24

## 목적

현재 MVP에 포함되는 것과 포함하지 않는 것을 분리한다.

## 현재 상태

- MVP는 운영형 Spoke 1개와 테스트베드형 Spoke 2개를 포함한 멀티 공장 관제 구조를 기준으로 정의한다.

## 범위

- 포함 범위
- 제외 범위
- 후속 확장 범위

## 상세 내용

## 포함 범위

- `factory-a` Safe-Edge 기준선 재구성
- `factory-b`, `factory-c` K3s 테스트베드형 Spoke
- AWS EKS Hub
- Tailscale 기반 Hub-Spoke 연결
- IoT Core -> S3 수집 경로
- Risk Score 기반 `안전 / 주의 / 위험` 표현
- 메인 대시보드
  - 공장별 위험 상태 카드
  - 센서 현황
  - 이상 시스템 목록
  - 최근 상태 변화 로그

## 제외 범위

- LLM 기반 일일 보고서 자동 생성
- event 기반 점수 반영
- 별도 이벤트 전용 파이프라인
- 공장별 상세 커스텀 정책 활성화
- 자동화된 장애/복구 리허설

## 후속 확장 범위

- event 입력 반영
- 분석 계층
- 공장 수 확대
- 세부 알람 정책
- 운영 자동화 고도화
